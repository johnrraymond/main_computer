# missing.txt

Confirmed missing or incomplete items for this `main_computer_test` snapshot.

This file is intentionally narrow. It lists gaps that are visible in the checked-in files, reproducible by static path checks, or still called out by the current `README.md`/`TODO.md`. It does not repeat old wishlist items that the current tree already partially implements.

## 1. Dev/control helper entry-point paths

The checked-in helper implementations live under `tools/`, not at the repository
root:

- `tools/dev-control.ps1`
- `tools/dev-chain-reset.py`
- `tools/dev-chain-flow.py`
- `tools/dev-chain-diagnosis.py`
- `tools/dev-chain-wallet-smoke-guide.py`
- `tools/dev-diagnosis.py`

Documentation and operator examples should use the `tools/...` paths directly.
Root-level compatibility wrappers are optional only if callers still need the old
entry points.

Current checked-in state:

- `tools/dev-control.ps1` exists.
- `tools/dev-chain-reset.py` exists.
- `tools/dev-chain-flow.py` exists.
- `tools/dev-chain-diagnosis.py` exists.

- `tools/smoke_fund_contract_preflight.py` exists and is the current faucet
  contract preflight smoke command.
- `dev-chain-smoke.py` is not present at the root or under `tools/` in this
  export.
- `dev-verify.py` is not present at the root or under `tools/` in this export.

Needed closure:

- Keep docs, tests, bootstrap output, and helper output aligned on the
  `tools/...` paths, or intentionally add root wrappers and test those wrappers.
- Remove obsolete `dev-chain-smoke.py` / `dev-verify.py` references, or replace
  them with the current concrete smoke/diagnosis commands.

## 2. Runtime deployment files are generated local state

The faucet/deployment runtime files are not expected to be committed:

- `runtime/deployments/current.json`
- `runtime/dev-chain/latest.json`
- `runtime/dev-chain/latest.env`

They are generated by the dev-chain reset/deploy flow, for example:

```powershell
python .\tools\dev-chain-reset.py --yes --run-id any-user-frobber-v1 --port-strategy auto
```

The repository `.gitignore` intentionally ignores broad `runtime/*` state while
allowing selected checked-in website runtime assets. This prevents local
deployment metadata, faucet/account material, and host-specific runtime files
from being exported as source.

Needed closure:

- If the viewport reports `Deployment runtime is missing or has no faucet
  account.`, run the dev-chain reset/deploy command from the repository root and
  restart the local app.
- Do not fix this by committing generated files under `runtime/deployments/` or
  `runtime/dev-chain/`.
- Keep documentation clear that `runtime/deployments/current.json` is an output
  of the dev deploy step, not a source-controlled input.

The tree contains separate terminal, executor, git, Aider, chat-console, task, and application routes. What remains missing is a single governed action contract across all of them.

Still missing or incomplete:

- One shared approval/preview contract for mutating terminal, agent, AI-generated, git, and application actions.

Needed closure:
