Busy-local overflow flow

  1. Try the local AI first.
  2. If local AI is busy, ask the hub for a compatible willing-worker count under this payment policy.
  3. Show the count to the user, not the workers' private minimum prices.
  4. If the user chooses Use Remote Worker, submit through the hub; if workers disappeared, create no request and hold no credits.
